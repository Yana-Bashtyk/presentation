Event Handling Models

  • Inline
  • Traditional
  • W3C model
  • Each event has an event handler - block of code that will be run when event fires.
  • There are several ways to assign a handler. Let’s see them, starting from the simplest one

Inline event handler

HTML with an attribute named on<event>.

  • Handler can be registered inline - using event handler HTML attributes, they usually have the next structure: onevent. The attribute value - JS code
  • It's the earliest method of registering event handlers found on the Web.

Traditional event handler

Assign a handler using a DOM property on<event>.

  • .onfocus and .onblur
  • window.onkeypress, window.onkeydown, window.onkeyup
  • .onmouseover, .onmouseout
  • ...
  • You can register handler - usage of DOM property onevent. There are many different event handler properties available.
  • onfocus, onblur - code will run when element is focused/ unfcocused (try pressing tab). Often used to display information about gow to fill form fields.
  • keypress - refers to a general press(button down and then up), keydown, keyup refer to just key down and key up parts
  • mouseover/out - the event is registered when mouse pointer is moved so it hover the element

W3C event handler

Use of special methods addEventListener and removeEventListener

  • The fundamental problem of the aforementioned ways to assign handlers – we can’t assign multiple handlers to one event.
  • Web-standard developers suggested an alternative way of managing handlers using special methods

element.addEventListener(event, handler[, phase])

  • Name of the event (string)
  • Function (function)
  • The “phase” for the handler to work (boolean)

element.removeEventListener (event, handler[, phase])

  • Listening for events can involve using the addEventListener function. This function takes three arguments.
  • Name - event you're interested in.
  • Handler itself - function thet will get called when event gets overheard It also provides access to the underlying event object as part of its arguments.
  • Boolean that declares whether the callback should be fired in capture phase.
  • We also have remove listener to use it when we want to remove event listener once they are no longer needed. Reference to the originally callback function.

Event Object

an argument passed to the handler.

  • type (string)
  • target (node)
  • currentTarget (node)
  • preventDefault (function)
  • stopPropagation (function)
  • When an event happens, the browser creates an event object, puts details into it and passes it as an argument to the handler.
  • We can use this object to access a wealth of information about the event that has occurred
  • type - the name of the event.
  • target - DOM node where the event originated
  • currentTarget - DOM node that event callback is firing on.
  • prevents default behaviour from occuring. for example, preventing a click event on an a element from loading a new page
  • stopPropagation - When this method gets called - your event stops his path and isn't allowed to continue forward either phase it involved in.

KeyBoard Events

  • keydown
  • keypress
  • keyup
  • There are three events that you will need to familiarize yourself with
  • keyup, keydown - clearly fired when you press down on a key or release a key.
  • keypress - fired only when you press down on a key that displys a char (letter, number)
  • They have the following order: -down, -press, -up

Argument properties:

  • keyCode
  • altKey, ctrlKey, shiftKey
  • metaKey
							
							window.addEventListener("keydown", checkKeyPressed, false);
							function checkKeyPressed(e) {
							
							if (e.keyCode == "65") {
							  alert("The 'a' key is pressed.");
							 }
							}
  • When an event handler that reacts to a keyboard event is called, a Keyboard event argument is passed in
  • keycode - Every key you press on your keyboard has a number associated with it. This read-only property returns that number.
  • charcode - exists on event arguments returned by the keypress event, and it contains the ASCII code for whatever character key you pressed
  • ctrl, alt, shift - return a true if these keys are pressed
  • meta - The Meta key is the Windows key on Windows keyboards and the Command key on Apple keyboards.

Capturing and Bubbling

  • Events don't start at the thing you trigger the event on.
  • When you initiate the event - events always start at the root
  • and slowly make its way all the way down until it hits a destination which is the element you clicked on. This phase called capturing.
  • After the event reaches the element it will make its way back to the root (origin) where it started. This is bubbling.

Third argument in addEvetnListener function specifies the phase

  • false => bubbling phase(default) 
    																	
								item.addEventListener("click", doSomething, false);
  • true => capturing phase 
    																	
								item.addEventListener("click", doSomething, true);
  • When you specify the event listener - as a third argument you specify the phase.
  • If it’s false (default), then the handler is set on the bubbling phase. If it’s true, then the handler is set on the capturing phase.
  • Well it's not very important if you are dealing with listening for the event on the element you're actually triggered an event on, but it's more important when dealing with intermediate elements on the path to your destination.

						 let selected;
						 const dots = document.querySelector('.navigation-dots-box')
						 dots.onclick = function(event) {
						   let target = event.target;
						   if (target.tagName != 'SPAN') return;
						   btnActive(target);

						   if (selected) {
						   selected.classList.remove('active')
						 }
						 selected = li;
						 selected.classList.add('active');
						 }
						</script>
  • Capturing and bubbling allow us to implement one of most powerful event handling patterns called event delegation.
  • We listen for event on the parent element instead of listening to each child.
  • In the handler we get event.target, see where the event actually happened and handle it.
  • Instead of assign an onclick handler to each span tag that contain numbers – we’ll setup the “catch-all” handler on div element. It will use event.target to get the clicked element and highlight it.
  • Such a code doesn’t care how many cells there are in the table. We can add/remove child elements dynamically at any time and the highlighting will still work.
